# Collection Management Tool
At the core of this tool we have vouchers which are bills against which collection have to be managed
Elements in this tools are
- users - 2 category of users
 - Distributor - He is the proprietory and have full owership
 - Salesman - collection installments against the voucher and submits to Distributor
- vouchers - this is a record of sales and agreement of payment
- beat - this is collection of shops to which goods have been sold on credit. Every credit sale is voucher

# what the tool should do
- Should have user based login and access to menu items
- A cli menu driven tool
- Data should be maintained in .csv files. 
- There should be staging area, merging of staging date to main data is only allowed for Distributor
